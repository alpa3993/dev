A Love Letter To Good Ole Git

Oh, the power of Git. With the help of Terminal I can use your magical commands to develop and stay on track of my "repos"

I commit to you. Or to many versions of you. With you, the power of my commands can take me to any time we shared together if I committed.

I know you can't resist my commands...

  • Make a file in a folder.
  • Mkdir newfile.html
  • cd newfile.html
  • ls am I where I need to be? Is this the right file? Is it you my love?
  • Begin repository code from Github

Now that I have you, what I want is only a command away.

  • Git init - Creates a repository and allows your directory to accept further Git commands.
  • Git help - With this command, you prompt me with FAQ and ways to solve issues I may have with you... always looking for solutions you are.
  • Git add(filename) - adds a file to staging environment
  • Git add . - adds all files in folder to staging environment
  • Git status - Checks the status of files connected to github account
  • Git commit -m "insert message" - Commits files from watch list to be tracked. "Insert a message to explain changes"
  • Git log - Checks your git log
  • Git config --global user.name - Allows you to change your Git username
  • Git config --global user.email - Allows you to change your Git email
  • Git checkout filename - This allows my commitment issues to kick in and jump back to the last committed version of your git files
  • Git branch - Checks which branch or repo you are currently in
  • Git checkout stringname - Allows user to switch to a new branch or repo
  • Git merge branchname - merges branhes together
  • Git checkout -b addnameofnewbranch - navigational command to a repo you would like to check out
  • Git push - Pushes committed files onto server
  • Git pull - pulls the most up to date file on Github
